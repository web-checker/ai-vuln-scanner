[![repo: auto-vuln-scanner](https://img.shields.io/badge/repo-auto--vuln--scanner-2ea44f?style=flat-square)](https://github.com/web-checker/auto-vuln-scanner)

[![shell: bash + PowerShell](https://img.shields.io/badge/shell-bash%20%2B%20PowerShell-89e051?style=flat-square)](#진단-영역--대상-호스트)
[![purpose: educational](https://img.shields.io/badge/purpose-educational-lightgrey?style=flat-square)](#라이선스--용도)

해당 대시보드는 KISA 주요정보통신기반시설 기술적 취약점 분석·평가, 클라우드 보안가이드(AWS) 자동진단 결과 CSV를 기반으로, AI 재판정과 자동화 스크립트 판정 교차검증을 수행하는 보안진단 대시보드입니다.

자동진단 스크립트가 생성한 로우데이터 CSV를 업로드하면 AI Agent가 주요정보통신기반시설 기술적 취약점 분석·평가 상세가이드, 클라우드 보안가이드(AWS) PDF와 필요 시 웹 검색을 참고하여 항목별 양호 / 취약 / N/A를 다시 판단합니다. 이후 자동화 스크립트 자체 판정과 AI 판정을 비교하여 불일치 항목을 확인하고, 최종 검토 결과를 Excel 보고서로 생성합니다.

> 로우데이터 CSV를 생성하는 자동진단 스크립트는 CHECKER의 [auto-vuln-scanner](https://github.com/web-checker/auto-vuln-scanner)를 활용합니다.

---

## 1. 프로젝트 개요

본 프로젝트는 자동 보안진단 결과를 단순히 표시하는 것이 아니라, 점검 증적과 기준 문서를 기반으로 AI가 한 번 더 판단하도록 설계된 진단 검토 도구입니다.
전체 흐름은 다음과 같습니다.

```text
자동진단 CSV 업로드
  → AI 독립 재판정
  → 자동화 스크립트 결과와 AI 결과 교차검증
  → 사용자 검토 및 확정
  → 자산별 진단 이력 저장
  → Excel 보고서 생성
```
자동진단 스크립트는 각 항목에 대해 1차 판정을 수행하고 CSV를 생성합니다.
해당 대시보드는 CSV에서 자동화 스크립트 판정값을 제외한 뒤, 판단기준과 점검내용만 AI Agent에 전달하여 독립적으로 재판정합니다.
재판정 이후에는 자동화 스크립트 결과와 AI 결과를 비교하여 불일치 항목을 표시하고, 사용자가 최종 결과를 검토·수정·확정할 수 있도록 지원합니다.

## 2. 주요 기능

### 2.1 CSV 기반 진단 결과 업로드

자동진단 스크립트가 생성한 로우데이터 CSV를 업로드하여 대시보드에서 항목별 진단 결과를 확인합니다.
CSV에는 다음과 같은 정보가 포함됩니다.

| 컬럼     | 설명                           |
| ------ | ---------------------------- |
| 항목코드   | 가이드 기준 항목 코드          |
| 분류     | 진단 항목의 분류                |
| 항목     | 진단 항목명                   |
| 판단기준   | 기준 문서의 양호 / 취약 판단 기준     |
| 결과     | 자동진단 스크립트의 1차 판정 결과      |
| 점검내용   | 점검에 사용된 명령 출력 원문         |
| 조치방법   | 취약 항목에 대한 조치 방법          |
| 진단대상   | 서버, 웹, WAS, DBMS 등 진단 대상 |
| 진단대상IP | 진단 대상 IP                 |
| 중요도    | 가이드 기준 항목 중요도                   |
| 점검파일   | 점검한 자동화 스크립트 / 설정 파일 |

시스템은 업로드된 CSV를 대시보드 표시용 데이터와 AI 판정용 데이터로 분리하여 처리합니다.

---

### 2.2 AI Agent 기반 독립 재판정

AI 판정 과정에서는 자동화 스크립트 자체 판정값인 결과 컬럼을 AI Agent에 전달하지 않습니다.
AI Agent에는 다음과 같은 정보만 전달합니다.

| 전달 항목 | 설명                       |
| ----- | ------------------------ |
| 항목코드  | 가이드 기준 항목 코드        |
| 항목    | 진단 항목명               |
| 판단기준  | 기준 문서의 양호 / 취약 판단 기준 |
| 진단대상  | 점검 대상 시스템            |
| 점검내용  | 자동진단 스크립트가 수집한 원문 증적 |
| 점검파일  | 점검한 파일 정보     |

자동화 스크립트 결과를 AI에 전달하지 않기 때문에, AI Agent는 점검 증적과 판단기준만을 기준으로 독립적인 판단을 수행합니다.
AI 재판정 결과는 이후 자동화 스크립트 결과와 비교하여 불일치 여부를 확인하는 데 사용합니다.

---

### 2.3 자동화 스크립트 결과와 AI 결과 교차검증

대시보드에서는 자동진단 스크립트의 1차 판정과 AI 재판정 결과를 함께 확인할 수 있습니다.

| 구분      | 설명                                   |
| ------- | ------------------------------------ |
| 자동화 스크립트 결과 | 자동진단 스크립트가 CSV에 기록한 1차 판정       |
| AI 결과   | AI Agent가 점검내용과 판단기준을 바탕으로 재판정한 결과 |
| 불일치 여부  | 자동화 스크립트 결과와 AI 결과가 다른 경우 표시        |

사용자는 불일치 항목을 중심으로 점검내용, 자동화 스크립트 근거, AI 판단 근거를 확인한 뒤 최종 결과를 확정할 수 있습니다.

---

### 2.4 참고 사항

모든 항목에 대해 처음부터 PDF 전체나 웹 검색을 사용하지 않습니다.
판단에 필요한 자료를 단계적으로 확장하여 토큰 사용량을 줄입니다.

```text
1단계: CSV의 판단기준과 점검내용만으로 즉시 판단
2단계: 판단이 부족한 경우 가이드 PDF의 해당 항목 섹션 참고
3단계: 버전, CVE, EOL 등 외부 확인이 필요한 경우 웹 검색 사용
```

대부분의 항목은 1단계에서 판단하며, 추가 기준 확인이 필요한 경우에만 PDF 또는 웹 검색을 사용합니다.

---

### 2.5 가이드 기반 조치방법 보완

CSV에 조치방법이 포함되어 있으면 해당 값을 그대로 사용합니다. 조치방법이 없는 경우, 결과가 취약인 항목은 가이드 PDF의 조치 방법 내용을 참고하여 보완합니다.
양호 또는 N/A 항목에는 조치방법을 입력하지 않습니다.

---

### 2.6 자산 및 진단 이력 관리

진단 결과는 진단대상 IP를 기준으로 로컬에 저장합니다.
같은 IP의 CSV를 다시 업로드하면 새로운 자산으로 생성하지 않고, 기존 자산의 새 진단기록으로 저장합니다.
이를 통해 동일 자산의 최초진단과 이행점검 결과를 비교할 수 있습니다.

저장 구조는 다음과 같습니다.

```text
config.DATA_DIR/
  assets.csv
  runs_index.csv
  runs/
    {asset_id}/
      {run_id}.csv
```

| 파일                             | 설명                      |
| ------------------------------ | ----------------------- |
| `assets.csv`                   | 진단대상 자산 목록 저장    |
| `runs_index.csv`               | 자산별 진단 실행 이력 저장   |
| `runs/{asset_id}/{run_id}.csv` | 특정 자산의 개별 진단 결과 저장 |

---

### 2.7 Excel 보고서 생성

확정된 진단 결과를 기준으로 Excel 보고서를 생성합니다.
보고서는 두 가지 버전으로 제작할 수 있습니다.

| 보고서    | 설명                         |
| ------ | -------------------------- |
| 최초 보고서 | 최초 진단 결과를 기준으로 생성합니다. (처음 파일 업로드 시 활용)      |
| 최종 보고서 | 최초진단과 이행점검 결과를 비교하여 생성합니다. (조치 후 이행 점검 진행 시 활용) |

Excel 보고서는 다음 5개 시트로 구성됩니다.

```text
1. 표지
2. 진단 대상
3. 요약 그래프
4. 요약 진단결과
5. 상세 결과
```

보고서 생성 단계에서는 LLM을 호출하지 않습니다.

### 2.8 AI 판정 엔진 (Claude)

AI 판정은 `claude_agent_sdk`를 통해 로컬에 설치된 `claude` CLI를 호출하여 수행합니다.
구독 인증을 사용하므로 별도의 API 키나 종량제 과금이 발생하지 않습니다.

| 항목      | 내용                                              |
| ------- | ----------------------------------------------- |
| 호출 방식   | `claude_agent_sdk`로 로컬 `claude` CLI 실행          |
| 인증 / 비용 | 구독 인증 사용, API 키 및 추가 과금 없음                       |
| 사용 모델   | 환경변수 `WAS_DIAG_MODEL` (기본값 `claude-opus-4-8`)   |
| 도구 사용   | 기본은 도구 없이 판단하고, 외부 확인이 필요할 때만 `WebSearch`를 허용    |
| 판정 규칙   | 시스템 프롬프트로 양호 / 취약 / N/A 기준과 출력 형식을 지정           |

항목별 판정은 점검 증적과 판단기준만으로 즉시 수행하며, 자료가 부족한 경우에만 가이드 PDF 섹션과 웹 검색을 단계적으로 사용합니다.

## 3. 판정 원칙

### 3.1 N/A 처리 기준

N/A는 해당 항목이 진단대상의 점검 대상이 아닐 때만 사용합니다.
증적이 부족하거나 확인이 어려운 경우에는 단순히 N/A로 처리하지 않고, 보수적으로 취약으로 판단합니다.

---

### 3.2 증적 부족 시 보수적 판단

점검내용만으로 양호 여부를 확인할 수 없거나, 판단에 필요한 증적이 부족한 경우에는 취약으로 판단합니다.
이는 실제 보안진단에서 확인되지 않은 항목을 양호로 처리하지 않기 위한 기준입니다.

## 4. 데이터 흐름

```text
CSV 업로드
  → preprocess.py
      - CSV 로드
      - AI 입력 항목 가공
      - 대시보드 표시용 테이블 생성

  → agent_sdk.py
      - Claude CLI 호출
      - 1단계 즉시판단
      - 필요 시 PDF 가이드 섹션 참고
      - 필요 시 웹 검색
      - 결과와 판단 근거 반환

  → dashboard.jsx / Detail.jsx
      - 자동화 스크립트 결과와 AI 결과 비교
      - 불일치 항목 표시
      - 사용자 검토 및 확정

  → store.py
      - 진단대상 IP 기준 자산 저장
      - 최초진단 / 이행점검 이력 관리

  → report.py
      - Excel 보고서 생성
      - 최초 보고서 또는 최종 비교 보고서 다운로드
```

## 5. AI 전달 여부(컬럼)

AI 판정에는 점검 증적과 기준만 전달합니다.
자동화 스크립트의 기존 판정 결과는 AI 입력에서 제외합니다.

| 구분      | 항목           |
| ------- | ------------ |
| 전달     | 항목코드         |
| 전달     | 항목           |
| 전달     | 판단기준         |
| 전달     | 진단대상         |
| 전달     | 점검내용         |
| 전달     | 점검파일         |
| 미전달 | 자동화 스크립트 결과 컬럼 |

자동화 스크립트 결과 컬럼은 AI 판정이 끝난 뒤, 교차검증 단계에서만 사용합니다.

## 10. 보안 및 운영 주의사항

업로드되는 CSV에는 실제 진단대상 IP, 시스템 설정, 점검 명령 출력 등이 포함될 수 있습니다.
따라서 다음 파일은 공개 저장소에 업로드하지 않는 것을 권장합니다.

```text
config.DATA_DIR/
assets.csv
runs_index.csv
runs/
업로드된 진단 결과 CSV
생성된 Excel 보고서
```

진단 결과와 보고서는 내부 검토용 산출물로 관리합니다.

## 11. 프로젝트 목적

해당 도구는 자동진단 스크립트의 결과를 그대로 신뢰하는 것이 아니라, 기준 문서와 실제 점검 증적을 기반으로 AI가 한 번 더 판단하도록 설계된 보안진단 검토 도구입니다.

이를 통해 다음 작업을 지원합니다.

* 자동진단 결과의 재검토
* 자동화 스크립트 판정과 AI 판정의 불일치 확인
* 진단대상 IP별 이력 관리
* 최초진단과 이행점검 비교
* Excel 기반 진단 보고서 생성
