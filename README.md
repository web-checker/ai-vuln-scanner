# ai-vuln-scanner (V-CHECKER)

`ai-vuln-scanner`는 KISA 주요정보통신기반시설 기술적 취약점 분석·평가 자동진단 결과 CSV를 기반으로, AI 재판정과 스크립트 판정 교차검증을 수행하는 보안진단 대시보드입니다.

자동진단 스크립트가 생성한 로우데이터 CSV를 업로드하면 Claude가 주통기 상세가이드 PDF와 필요 시 웹 검색을 참고하여 항목별 `양호 / 취약 / N/A`를 다시 판단합니다. 이후 스크립트 자체 판정과 AI 판정을 비교하여 불일치 항목을 확인하고, 최종 검토 결과를 Excel 보고서로 생성합니다.

로우데이터 CSV를 생성하는 자동진단 스크립트는 [auto-vuln-scanner](https://github.com/web-checker/auto-vuln-scanner)를 참고합니다.

---

## 1. 프로젝트 개요

본 프로젝트는 자동 보안진단 결과를 단순히 표시하는 것이 아니라, 점검 증적과 기준 문서를 기반으로 AI가 한 번 더 판단하도록 설계된 진단 검토 도구입니다.

자동진단 스크립트는 각 항목에 대해 1차 판정을 수행하고 CSV를 생성합니다.
V-CHECKER는 해당 CSV에서 스크립트 판정값을 제외한 뒤, 판단기준과 점검내용만 Claude에 전달하여 독립적으로 재판정합니다.

재판정 이후에는 스크립트 결과와 AI 결과를 비교하여 불일치 항목을 표시하고, 사용자가 최종 결과를 검토·수정·확정할 수 있도록 지원합니다.

전체 흐름은 다음과 같습니다.

```text
자동진단 CSV 업로드
  → AI 독립 재판정
  → 스크립트 결과와 AI 결과 교차검증
  → 사용자 검토 및 확정
  → 자산별 진단 이력 저장
  → Excel 보고서 생성
```

## 2. 주요 기능

### 2.1 CSV 기반 진단 결과 업로드

자동진단 스크립트가 생성한 로우데이터 CSV를 업로드하여 대시보드에서 항목별 진단 결과를 확인합니다.

CSV에는 다음과 같은 정보가 포함됩니다.

| 컬럼     | 설명                           |
| ------ | ---------------------------- |
| 항목코드   | KISA 진단 항목 코드입니다.            |
| 분류     | 진단 항목의 분류입니다.                |
| 항목     | 진단 항목명입니다.                   |
| 판단기준   | 기준 문서의 양호 / 취약 판단 기준입니다.     |
| 결과     | 자동진단 스크립트의 1차 판정 결과입니다.      |
| 점검내용   | 점검에 사용된 명령 출력 원문입니다.         |
| 조치방법   | 취약 항목에 대한 조치 방법입니다.          |
| 진단대상   | 서버, 웹, WAS, DBMS 등 진단 대상입니다. |
| 진단대상IP | 진단 대상 IP입니다.                 |
| 중요도    | 항목 중요도입니다.                   |
| 점검파일   | 해당 항목을 점검한 스크립트 또는 설정 파일입니다. |

시스템은 업로드된 CSV를 대시보드 표시용 데이터와 AI 판정용 데이터로 분리하여 처리합니다.

---

### 2.2 Claude 기반 독립 재판정

AI 판정 과정에서는 스크립트 자체 판정값인 `결과` 컬럼을 Claude에 전달하지 않습니다.

Claude에는 다음 정보만 전달합니다.

| 전달 항목 | 설명                       |
| ----- | ------------------------ |
| 항목코드  | KISA 진단 항목 코드입니다.        |
| 항목    | 진단 항목명입니다.               |
| 판단기준  | 기준 문서의 양호 / 취약 판단 기준입니다. |
| 진단대상  | 점검 대상 시스템입니다.            |
| 점검내용  | 자동진단 스크립트가 수집한 원문 증적입니다. |
| 점검파일  | 해당 항목을 점검한 파일 정보입니다.     |

스크립트 결과를 AI에 전달하지 않기 때문에, Claude는 점검 증적과 판단기준만을 기준으로 독립적인 판단을 수행합니다.

AI 재판정 결과는 이후 스크립트 결과와 비교하여 불일치 여부를 확인하는 데 사용합니다.

---

### 2.3 스크립트 결과와 AI 결과 교차검증

대시보드에서는 자동진단 스크립트의 1차 판정과 AI 재판정 결과를 함께 확인할 수 있습니다.

| 구분      | 설명                                   |
| ------- | ------------------------------------ |
| 스크립트 결과 | 자동진단 스크립트가 CSV에 기록한 1차 판정입니다.        |
| AI 결과   | Claude가 점검내용과 판단기준을 바탕으로 재판정한 결과입니다. |
| 불일치 여부  | 스크립트 결과와 AI 결과가 다른 경우 표시됩니다.         |

사용자는 불일치 항목을 중심으로 점검내용, 스크립트 근거, AI 판단 근거를 확인한 뒤 최종 결과를 확정할 수 있습니다.

---

### 2.4 3단계 토큰 절감 판정

모든 항목에 대해 처음부터 PDF 전체나 웹 검색을 사용하지 않습니다.
판단에 필요한 자료를 단계적으로 확장하여 토큰 사용량을 줄입니다.

```text
1단계: CSV의 판단기준과 점검내용만으로 즉시 판단합니다.
2단계: 판단이 부족한 경우 주통기 상세가이드 PDF의 해당 항목 섹션을 참고합니다.
3단계: 버전, CVE, EOL 등 외부 확인이 필요한 경우 웹 검색을 사용합니다.
```

대부분의 항목은 1단계에서 판단하며, 추가 기준 확인이 필요한 경우에만 PDF 또는 웹 검색을 사용합니다.

---

### 2.5 주통기 상세가이드 기반 조치방법 보완

CSV에 조치방법이 포함되어 있으면 해당 값을 그대로 사용합니다.

CSV에 조치방법이 없고 결과가 `취약`인 항목은 주통기 상세가이드 PDF의 `조치 방법` 내용을 참고하여 보완합니다.

`양호` 또는 `N/A` 항목에는 조치방법을 입력하지 않습니다.

---

### 2.6 자산 및 진단 이력 관리

진단 결과는 진단대상 IP를 기준으로 로컬에 저장합니다.

같은 IP의 CSV를 다시 업로드하면 새로운 자산으로 생성하지 않고, 기존 자산의 새 진단기록으로 저장합니다.
이를 통해 동일 자산의 최초진단과 이행점검 결과를 비교할 수 있습니다.

저장 구조는 다음과 같습니다.

```text
config.DATA_DIR/
  assets.csv
  runs_index.csv
  runs/
    {asset_id}/
      {run_id}.csv
```

| 파일                             | 설명                      |
| ------------------------------ | ----------------------- |
| `assets.csv`                   | 진단대상 자산 목록을 저장합니다.      |
| `runs_index.csv`               | 자산별 진단 실행 이력을 저장합니다.    |
| `runs/{asset_id}/{run_id}.csv` | 특정 자산의 개별 진단 결과를 저장합니다. |

---

### 2.7 Excel 보고서 생성

확정된 진단 결과를 기준으로 Excel 보고서를 생성합니다.

보고서는 두 가지 형태를 지원합니다.

| 보고서    | 설명                         |
| ------ | -------------------------- |
| 최초 보고서 | 단일 진단 결과를 기준으로 생성합니다.      |
| 최종 보고서 | 최초진단과 이행점검 결과를 비교하여 생성합니다. |

Excel 보고서는 다음 5개 시트로 구성됩니다.

```text
1. 표지
2. 진단 대상
3. 요약 그래프
4. 요약 진단결과
5. 상세 결과
```

요약 그래프에는 방사형 차트, 원형 차트, 막대 차트가 포함됩니다.
보고서 생성 단계에서는 LLM을 호출하지 않습니다.

## 3. 판정 원칙

### 3.1 N/A 처리 기준

`N/A`는 해당 항목이 진단대상의 점검 대상이 아닐 때만 사용합니다.

증적이 부족하거나 확인이 어려운 경우에는 단순히 `N/A`로 처리하지 않고, 보수적으로 `취약`으로 판단합니다.

---

### 3.2 증적 부족 시 보수적 판단

점검내용만으로 양호 여부를 확인할 수 없거나, 판단에 필요한 증적이 부족한 경우에는 `취약`으로 판단합니다.

이는 실제 보안진단에서 확인되지 않은 항목을 양호로 처리하지 않기 위한 기준입니다.

---

### 3.3 열거형 항목 판단

숨김파일, 계정, SUID 파일처럼 목록을 확인하는 항목은 정상 항목이 나열되어 있다는 이유만으로 `양호` 처리하지 않습니다.

화이트리스트 또는 기준값과 비교하여 비표준 항목이 있는지 확인하고, 이상치가 하나라도 있으면 `취약`으로 판단합니다.

---

### 3.4 권한 비교 방식

파일 권한은 단순 숫자 비교가 아니라 `owner / group / other` 권한을 각각 비교합니다.

기준보다 과도한 권한이 부여된 경우 취약으로 판단합니다.

```text
      - 불일치 항목 표시
      - 사용자 검토 및 확정

  → store.py
      - 진단대상 IP 기준 자산 저장
      - 최초진단 / 이행점검 이력 관리

      - Excel 보고서 생성

```

## 5. AI에 전달하는 값과 전달하지 않는 값

AI 판정에는 점검 증적과 기준만 전달합니다.
스크립트의 기존 판정 결과는 AI 입력에서 제외합니다.

| 구분      | 항목           |
| ------- | ------------ |
| 전달함     | 항목코드         |
| 전달함     | 항목           |
| 전달함     | 판단기준         |
| 전달함     | 진단대상         |
| 전달함     | 점검내용         |
| 전달함     | 점검파일         |
| 전달하지 않음 | 스크립트 `결과` 컬럼 |

스크립트 `결과` 컬럼은 AI 판정이 끝난 뒤, 교차검증 단계에서만 사용합니다.

## 7. 실행 방법

### 7.1 사전 준비

다음 환경이 필요합니다.

* Python 백엔드 실행 환경
* Node.js LTS
* Claude CLI 설치 및 구독 로그인
* 주통기 상세가이드 PDF
* 자동진단 결과 CSV

---

### 7.2 Python 의존성 설치

```powershell
pip install -r requirements.txt
```

---

### 7.3 Claude CLI 로그인

Claude CLI를 설치한 뒤 최초 1회 로그인합니다.

```powershell
claude
/login
```

본 프로젝트는 로컬 Claude CLI를 호출하는 방식으로 동작합니다.
별도의 API 키를 사용하지 않고 Claude CLI 로그인 환경을 기준으로 실행합니다.

모델은 `WAS_DIAG_MODEL` 환경변수로 지정할 수 있으며, 기본값은 `claude-opus-4-8`입니다.

---

### 7.4 운영 모드 실행

```powershell
run-app.bat
```

운영 모드는 프론트엔드를 빌드한 뒤 FastAPI 서버에서 정적 파일을 함께 서빙합니다.

실행 후 브라우저에서 다음 주소로 접속합니다.

```text
http://localhost:8600
```

---

### 7.5 개발 모드 실행

```powershell
run-dev.bat
```

개발 모드는 백엔드와 프론트엔드를 분리하여 실행합니다.

| 구분          | 주소                      |
| ----------- | ----------------------- |
| FastAPI 백엔드 | `http://localhost:8600` |
| Vite 프론트엔드  | `http://localhost:5173` |

---

### 7.6 단건 판정 확인

UI 없이 특정 항목의 AI 판정을 확인하려면 다음 명령을 사용합니다.

```powershell
python smoke_test.py WEB-16
```

## 8. 사용 흐름

```text
1. 자동진단 결과 CSV를 업로드합니다.
2. 진단 종류를 선택합니다.
3. AI 교차 진단을 실행합니다.
4. 항목별 스크립트 결과와 AI 결과를 비교합니다.
5. 불일치 항목의 판단 근거와 점검내용을 확인합니다.
6. 사용자가 최종 결과를 검토하고 확정합니다.
7. 진단 결과를 자산에 저장합니다.
8. Excel 보고서를 다운로드합니다.
```

## 11. 보안 및 운영 주의사항

업로드되는 CSV에는 실제 진단대상 IP, 시스템 설정, 점검 명령 출력 등이 포함될 수 있습니다.

따라서 다음 파일은 공개 저장소에 업로드하지 않는 것을 권장합니다.

```text
config.DATA_DIR/
assets.csv
runs_index.csv
runs/
업로드된 진단 결과 CSV
생성된 Excel 보고서
```

진단 결과와 보고서는 내부 검토용 산출물로 관리합니다.

## 12. 프로젝트 목적

V-CHECKER는 자동진단 스크립트의 결과를 그대로 신뢰하는 것이 아니라, 기준 문서와 실제 점검 증적을 기반으로 AI가 한 번 더 판단하도록 설계된 보안진단 검토 도구입니다.

이를 통해 다음 작업을 지원합니다.

* 자동진단 결과의 재검토
* 스크립트 판정과 AI 판정의 불일치 확인
* 진단대상 IP별 이력 관리
* 최초진단과 이행점검 비교
* Excel 기반 진단 보고서 생성
