  - **표기가 안 되던 진짜 이유 2가지:**
    1. 실제 실행 중이던 (ai_jy) 버전은 구버전이라 **대시보드/엑셀에 조치방법 열 자체가 없었음**.
       이 폴더(git_tmp2)에는 열이 추가돼 있음 → 이 폴더로 배포하면 해결.
    2. 조치방법을 **"취약" 항목에만** 노출하도록 막혀 있어, 양호/N/A 항목의 CSV 조치방법 값이 공란 처리됨.
- **조치:** CSV `조치방법` 열 값을 **결과(양호/취약/N/A) 구분 없이 항상 노출**하도록 변경.
  - `frontend/src/App.jsx`, `frontend/src/RunDetail.jsx`: `isVuln(it) ? remediation : ''` → `it.remediation` 항상 표기.
  - `ai_app/report.py`: `build_report_df`, `build_report_df_from_run`의 `show_remed`(취약 게이팅) 제거.
  - 검증: 저장된 Run에서 WEB-03(양호) 항목도 조치방법 값이 보고서/엑셀에 출력 확인.

  자산 1개 + 진단기록 13건으로 누적됨(같은 IP 재업로드 = 새 자산이 아니라 새 Run).
