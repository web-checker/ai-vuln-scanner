## "항목별 양호 비율" — 도메인 1~2개일 때 얇은 속빈 막대로 폴백

- **증상:** 이행점검/최초진단 산출물의 `2-1. 요약결과(그래프)` 시트 1번 블록
  "항목별 양호 비율"에서, 진단 도메인이 **1개(예: 계정 관리만)** 이면 값이 100%여도
  그래프에 아무것도 안 보임(축/세로선만 남음). 표에는 `100.0%`로 값이 정상 표기됨.
- **원인:** 이 블록은 **방사형(radar) 차트**인데, 방사형은 **꼭짓점(도메인)이 최소 3개**
  있어야 면(다각형)을 그림. 도메인이 1개면 축 하나, 2개면 선 하나로 degenerate → 안 보임.
  데이터 문제가 아니라 방사형 차트의 성질.
- **결정(설계 논의):**
  - 방사형은 팀원 작업이라 **3개 이상에서는 그대로 유지**(정상 데이터는 대부분 도메인 여러 개).
  - 1~2개일 때만 대체. 막대는 부피감이 커 방사형과 느낌이 달라지는 점을 고려해,
    **채움 없는(noFill) 얇은 파란 윤곽 막대**로 그려 방사형의 "가는 선" 느낌에 가깝게 맞춤.
  - 가짜 도메인·꺾은선(추이용이라 취지 불일치)은 배제.
- **변경 파일:** `ai_app/report.py` — `_sheet_graph()` 블록1
  - `n = len(groups)` 분기 추가: `n >= 3` → `RadarChart`(기존 그대로),
    `n < 3` → `BarChart(type="col")` + `gapWidth=400`(얇게) + `noFill`(속 빈 막대)
    + 방사형과 동일한 파랑 `4472C4` 얇은 윤곽선.
  - y축 스케일 0~100%·`0%` 포맷·회색 축 라벨은 양쪽 공통 유지(방사형과 스케일 일치).
- **검증:** 도메인 1·2·3개 각각 `_build_workbook` → `2-1` 시트 블록1 차트 종류 확인.
  - 1개 → `BarChart`(noFill=True), 2개 → `BarChart`(noFill=True), 3개 → `RadarChart`.
  - 세 경우 모두 .xlsx 저장(직렬화) 정상.

## 후속 참고

  (`noFill` + 윤곽선 렌더는 뷰어별 편차가 있을 수 있음).

  이는 값(0)이 맞는 정상 동작으로 이번 변경 범위 아님.
