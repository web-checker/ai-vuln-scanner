## 문서번호 구분코드 — 최초진단=VA / 이행점검=CA (자동 분류)

- **요청:** 문서번호 형식 `팀명-VA-연도-번호`에서 구분코드(`VA`)를
  **최초진단일 때만 VA**, **이행점검일 때는 CA**로 작성되게.
- **기존 상태:** `REPORT_META["문서번호"] = "CHECKBANG-VA-2026-001"` 로 전체가 하드코딩되어
  진단 종류와 무관하게 항상 `VA`로 출력됐음(표지 0번 시트 우상단 메타 박스).
- **변경 파일:** `ai_app/report.py`
  - `_doc_number(is_followup)` 헬퍼 추가 — 문서번호를 `-`로 분해해 **2번째 세그먼트(구분코드)만**
    `CA`(이행) / `VA`(최초)로 교체. 팀명·연도·일련번호(`CHECKBANG`·`2026`·`001`)는 원본 유지.
  - `_build_workbook` → `_sheet_cover(..., is_followup=compare_map is not None)` 로 진단 종류 전달.
  - `_sheet_cover(ws, S, is_followup=False)`: 문서번호를 `_doc_number(is_followup)`로 산출해 표기.
- **분류 기준(기존 신호 재사용):** `compare_map` 유무.
  - 최초진단 경로(`build_xlsx_report` / `build_xlsx_from_run`) → `compare_map` 없음 → **VA**
  - 이행점검 경로(`build_final_xlsx`, base↔target 병합) → `compare_map` 있음 → **CA**
  - 별도의 `종류` 필드를 새로 만들지 않고, 이미 존재하던 이 구분 신호를 그대로 사용.
- **검증:** `report._doc_number(False)` → `CHECKBANG-VA-2026-001`,
  `report._doc_number(True)` → `CHECKBANG-CA-2026-001` 확인.

## 후속 참고

- `_sheet_cover`의 `is_followup=False`는 방어적 기본값(플래그 미전달 시 최초진단으로 폴백).
  실제 호출부(`_build_workbook`)는 항상 정확한 값을 넘김.
- 연도·일련번호는 이번 범위가 아니라 그대로 둠(자동 증가 필요 시 별도 요청).
- 향후 "이행점검을 `compare_map` 없이 단일 보고서로 뽑는" 예외 경로가 생기면 이 분류 기준을 재검토해야 함.
