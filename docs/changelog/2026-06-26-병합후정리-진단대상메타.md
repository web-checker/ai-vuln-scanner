## 1. 병합 후 2차 정리 (커밋 `6f885ee`)

hj·jy 가 main 에 얹은 신규 기능(보고서 확장·ReportTab·사이드바 개편·페이지네이션)을
전수 점검해 회귀·버그·데드코드를 정리.

### 보안·버그
- **path-traversal 회귀 차단** — `get_saved_report`·`get_run`·`save_run_report`·
  `update_run_kind`·`delete_run` 에 `_safe_run_id` 누락 → 임의 `.html` 읽기/쓰기 가능했음.
  (백슬래시 `%5c`·점 경로 이탈은 라우팅이 못 막아 핸들러까지 도달 → 가드로 400 차단)
- **비교 탭 필터 깨짐** — 버튼 라벨('조치 완료'·'신규 취약'·'양호 유지')이 백엔드 상태값
  ('개선'·'악화'·'양호유지'(공백차이))과 불일치 → 5개 중 3개 필터가 항상 0건.
  `ui.CMP_FILTERS`(라벨↔값 매핑)로 수정.
- **최종 보고서 항목 누락** — `build_final_report_rows` 가 base 기준 루프라 이행점검에서
  새로 추가된 항목이 누락. base·target 코드 합집합(`_union_codes`)으로 수정.
- judge 스트림 `_flush` 를 `try/finally` 로 — 클라이언트 단절 시에도 부분결과 저장.
- `store.compare` 결과값 strip — 보고서 pill 색상 오표기('취약 '→na) 방지.
- `_sheet_graph` 블록 간격 STEP 동적화 — 분류 수가 많을 때 표/차트 겹침 방지.

### 데드코드
- 죽은 API 4개(`setRunKind`·`saveRunReportHtml`·`saveCompareReport`·`savedReportUrl`),
  `kindPage` 상태, Sidebar 미사용 prop 4개, `_label_reason` 래퍼, 죽은 CSS
  (`.kind-*`·자산트리 `.tree-twist/badge/del/empty/pager`·`.vcmp-*` 모디파이어).

### 구조
- 가이드 조치방법 폴백 단일화(`main._guide_remed` → `report._guide_remediation`).
- `reportKind 'first'/'final'` 상수화, `_XLSX_MEDIA` 상수 일관 적용.

> 주의(감사 교훈): `runKind`/`setRunKind` 는 jy 가 추가한 사이드바 토글이 재사용 중이라
> 정적 "데드" 판정을 grep 으로 재확인 후 **보존**함.

## 2. 진단 대상 시트 메타 + 라벨 (커밋 `eeb83ba`)

### 보고서 "1. 진단 대상" 시트에 HOSTNAME·버전정보 삽입
- CSV 선택 컬럼 `HOSTNAME`·`버전정보` 추가(`config`) — **필수 아님**, 옛 CSV 하위호환(없으면 공란/`-`).
- **CSV 경량화**: 스크립트가 전 행에 반복하지 말고 **첫 데이터 행에만** 채우면 됨
  — 보고서가 '첫 비어있지 않은 값'을 읽으므로 중복 0 으로 동일 동작.
- Run CSV(`RUN_COLUMNS`)에도 보존 → **세션·저장 Run·이행점검** 보고서 모두 표기.
  (`store.build_run_df` / `report._host_meta` / `report.build_xlsx_from_run`)

### 보고서 서브탭 라벨
- 최초 보고서 → **최초 진단**, 최종 보고서 → **이행 점검**
  (사이드바 · 상단 경로(breadcrumb) · ReportTab 카드 제목 일괄).
