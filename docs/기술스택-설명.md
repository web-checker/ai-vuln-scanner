# 기술 스택 쉽게 알아보기 (중·고등학생 눈높이)

이 프로젝트(**V-CHECKER**)는 *"보안 점검 결과(CSV)를 AI가 다시 읽고 양호/취약을 재판정해서 보고서를 만들어 주는 웹 프로그램"* 이다.
아래는 이 프로그램을 만드는 데 쓴 부품(기술)들을 하나씩, 비유를 곁들여 설명한 것이다.

> 전체 그림 한 줄 요약
> **브라우저 화면(React)** ↔ **중간 서버(FastAPI)** ↔ **AI 판정(Claude)** + **데이터/문서 처리(pandas·openpyxl·pypdf)**

---

## 🤖 AI — "사람 대신 판단하는 두뇌"

### Claude Opus 4.8 (`claude-opus-4-8`)
- **무엇?** Anthropic이라는 회사가 만든 **대화형 인공지능(LLM, 거대 언어 모델)**. ChatGPT 같은 종류라고 보면 된다.
- **비유:** 보안 지식이 아주 많은 *베테랑 점검 전문가*를 채팅으로 부르는 것.
- **왜 이걸?** "Opus"는 그 가족 중 **가장 똑똑한 모델**이라 보안 판단처럼 실수하면 안 되는 일에 쓴다.
- **여기서 하는 일:** 점검 결과(예: "root 원격 접속 허용됨")를 읽고 **양호 / 취약 / N/A** 중 하나로 판정하고, 그 이유를 적어 준다.

### claude-agent-sdk — "AI에게 일을 시키는 리모컨"
- **무엇?** 프로그램(파이썬 코드) 안에서 Claude를 **자동으로 불러 쓰게** 해 주는 도구 묶음(SDK = Software Development Kit, 개발 도구 모음).
- **비유:** 사람이 직접 채팅창에 타이핑하는 대신, **버튼 하나로 AI에게 질문을 보내고 답을 받아오는 리모컨**.
- **왜 이걸?** 점검 항목이 수십~수백 개라 사람이 일일이 물어볼 수 없으니, 코드가 **자동으로 한 항목씩 AI에게 물어보게** 한다.

### Claude Code CLI (로컬 · 구독 인증 · API 키 불필요)
- **CLI**란? Command Line Interface, 즉 **검은 화면(터미널)에 명령어를 쳐서 쓰는 프로그램**.
- **로컬(local):** 내 컴퓨터에 깔린 프로그램을 쓴다는 뜻.
- **핵심 장점 — "API 키 불필요":**
  - 보통 AI를 코드에서 쓰려면 **API 키**(= 사용할 때마다 **돈이 빠져나가는 출입증**)가 필요하다.
  - 이 프로젝트는 그 대신 이미 **구독(정액제)** 으로 결제된 Claude Code를 빌려 쓴다. → **추가 종량제 요금 없이** 구독 사용량에서만 차감된다. 학습·실습용으로 부담이 적다.

### WebSearch — "AI의 인터넷 검색 기능 (3단계에서만)"
- **무엇?** AI가 **최신 정보를 인터넷에서 직접 찾아보는** 기능.
- **왜 필요?** AI는 학습한 시점까지의 지식만 안다. 그런데 보안은 *"이 소프트웨어 버전이 지금도 안전한가? 새 취약점(CVE)이 있나?"* 처럼 **최신 정보**가 필요할 때가 있다.
- **"3단계"의 의미 (토큰 = AI 사용 비용 절약 전략):**
  1. **1단계** — 점검 결과만 보고 바로 판단 (대부분 여기서 끝, 비용 0)
  2. **2단계** — 애매하면 KISA 가이드 PDF의 해당 부분만 더 읽고 판단
  3. **3단계** — 그래도 애매하면(버전·CVE 등) 그때만 **WebSearch**로 검색
  - → 쉬운 건 빨리, 어려운 것만 깊게. **불필요한 검색·비용을 아끼는 구조**.

## 🖥️ Backend — "화면 뒤에서 실제 일을 처리하는 부엌"

> **Backend(백엔드)** = 사용자 눈에 안 보이는 **서버 쪽**. 주문(요청)을 받아 요리(처리)해서 내보낸다.
> (반대로 눈에 보이는 화면은 Frontend, 뒤에 나온다.)

### Python 3.12 — "사람 말에 가까운 프로그래밍 언어"
- **무엇?** 배우기 쉽고 읽기 편한 **프로그래밍 언어**. 3.12는 버전(버전 숫자가 클수록 최신).
- **비유:** 요리로 치면 **레시피를 쓰는 언어**. 영어로도 쓸 수 있고 한국어로도 쓸 수 있듯, 프로그램도 여러 언어로 짤 수 있는데 그중 파이썬을 골랐다.
- **왜 이걸?** AI·데이터 처리 분야에서 **가장 널리 쓰여서** 관련 도구(라이브러리)가 풍부하다.

### FastAPI — "주문을 받는 창구(REST API)"
- **API**란? Application Programming Interface. **프로그램끼리 정해진 규칙으로 대화하는 약속**.
  - 비유: 식당 **메뉴판**. "1번 시키면 김밥이 나온다"처럼, "이 주소로 요청하면 이 데이터가 온다"는 약속.
- **REST API:** 그 약속을 만드는 **널리 쓰이는 표준 방식**. (예: `/api/upload`로 파일 보내면 처리해 줌)
- **FastAPI:** 파이썬으로 이 창구를 **빠르고 쉽게** 만들게 해 주는 도구. 이름처럼 빠르다(Fast).
- **여기서 하는 일:** 화면(React)이 보내는 *"이 CSV 판정해 줘", "보고서 줘"* 같은 요청을 받아 처리하고 돌려준다.

### Uvicorn — "창구에 손님을 실제로 받는 직원(ASGI 서버)"
- **무엇?** FastAPI로 만든 프로그램을 **실제로 켜서 인터넷 요청을 받게** 해 주는 프로그램(서버).
- **비유:** FastAPI가 **메뉴판과 주방 설계도**라면, Uvicorn은 **실제로 문을 열고 손님을 받는 직원**.
- **ASGI:** 파이썬 웹 프로그램이 요청을 주고받는 **현대식 표준 규격**. (예전 방식보다 동시에 여러 손님을 잘 받는다.)
- 그래서 `run-app.bat`을 실행하면 Uvicorn이 켜지고 `http://localhost:8600`으로 접속할 수 있게 된다.

### python-multipart — "파일 업로드를 풀어 주는 도구"
- **무엇?** 웹에서 **파일을 올릴 때** 그 데이터를 서버가 알아먹게 풀어 주는 작은 부품.
- **왜 필요?** 사용자가 점검 결과 **CSV 파일을 업로드**하는데, 그 파일 데이터를 서버가 받아 처리하려면 이 도구가 있어야 한다.
- **비유:** 택배(파일)가 오면 **포장을 뜯어 내용물을 꺼내 주는 사람**.

## 🎨 Frontend — "사람이 보고 클릭하는 화면"

> **Frontend(프런트엔드)** = 사용자가 **눈으로 보고 손으로 클릭하는** 부분. 식당으로 치면 **손님이 앉는 홀과 메뉴판**.

### React 18.3 — "화면을 블록처럼 조립하는 도구"
- **무엇?** 페이스북(Meta)이 만든, **웹 화면을 만드는 인기 도구(라이브러리)**.
- **핵심 아이디어 — 컴포넌트(부품):** 화면을 **레고 블록처럼 작은 조각**으로 나눠 만들고, 데이터가 바뀌면 **그 부분만 자동으로 다시 그린다**.
  - 비유: 점수가 바뀌면 **전광판 전체가 아니라 점수 칸만** 바뀌는 것.
- **여기서:** 업로드 화면, 판정 결과 표, 차트, 자산 관리 화면 같은 걸 React 부품으로 만들었다.
  (예: 우리가 만든 `dashboard.jsx`는 "요약 차트 + 결과 표"를 **여러 화면이 공유하는 부품**으로 묶은 것.)

### Vite 5.4 — "코드를 브라우저용으로 묶어 주는 빠른 포장 기계"
- **무엇?** 우리가 짠 여러 개의 코드 파일(JSX, CSS 등)을 **브라우저가 이해하는 형태로 합치고 최적화**해 주는 **빌드 도구**.
- **비유:** 흩어진 **재료를 한 상자에 깔끔히 포장**해 주는 기계. ("build"가 바로 이 포장 과정.)
- **왜 이걸?** 이름처럼 **매우 빨라서**(Vite는 프랑스어로 "빠르다") 개발 중 수정하면 화면이 거의 즉시 바뀐다.

### JavaScript(JSX) · CSS
- **JavaScript:** **웹페이지를 움직이게** 하는 언어. (버튼 클릭, 데이터 표시 등 동작 담당.)
- **JSX:** JavaScript 안에 **HTML 화면 모양을 그대로 써넣는** 특별한 문법. React에서 화면을 직관적으로 짜게 해 준다.
  - 예: `<button>판정 시작</button>` 처럼 코드 안에 화면이 보인다.
- **CSS:** 화면의 **색·글꼴·배치 등 "꾸미기"** 를 담당. (HTML이 뼈대라면 CSS는 옷과 화장.)

## 📊 데이터 · 문서 처리 — "자료를 다루고 보고서를 만드는 도구"

### pandas — "코드로 다루는 엑셀 표"
- **무엇?** 파이썬에서 **표 형태 데이터(행·열)** 를 자유자재로 다루는 도구.
- **비유:** **엑셀을 코드로 조작**한다고 보면 된다. 필터·정렬·합계·그룹 묶기를 몇 줄로 한다.
- **여기서:** 업로드된 점검 결과 CSV를 **읽고, 정리하고, 양호/취약 개수를 집계**하는 데 쓴다.

### openpyxl — "엑셀 보고서를 코드로 만드는 도구"
- **무엇?** 파이썬으로 **엑셀 파일(.xlsx)을 직접 만들고 꾸미는** 도구.
- **여기서:** 최종 **진단 보고서를 엑셀로 출력**한다. 단순 표가 아니라 **수식(자동 합계)·여러 시트·차트**까지 넣은 보기 좋은 보고서를 만든다.
- **비유:** 사람이 엑셀에서 직접 하는 *셀 색칠·합계 함수·차트 삽입*을 **코드가 자동으로** 해 준다.

### pypdf — "PDF에서 글자를 뽑아내는 도구"
- **무엇?** **PDF 문서에서 텍스트(글자)를 추출**하는 도구.
- **여기서:** KISA(한국인터넷진흥원)가 낸 **취약점 점검 가이드 PDF**에서, AI가 판단에 참고할 **해당 항목 설명 부분만 글자로 뽑아** AI에게 건넨다.
- **왜 일부만?** PDF 전체를 통째로 AI에게 주면 **비용(토큰)이 크게** 든다. 필요한 페이지의 글자만 뽑아 **싸고 빠르게** 처리한다.

## 🛠️ 기타 — "협업과 데이터 전달 방식"

### Git / GitHub — "코드의 타임머신 + 공동 작업 공간"
- **Git:** 코드의 **변경 이력을 저장하는 도구**. 언제든 과거로 되돌리고, 누가 뭘 바꿨는지 추적한다.
  - 비유: 게임의 **세이브 포인트** + 글의 **수정 기록**.
- **GitHub:** 그 Git 저장소를 **인터넷에 올려 여러 명이 함께 작업**하는 사이트.
  - 비유: 여러 명이 같이 쓰는 **구글 문서**의 코드 버전.
- **여기서:** 팀원들이 각자 브랜치(작업 가지)에서 만들고, **main(본 줄기)** 에 합쳐 함께 개발한다.
  (예: 보안 점검을 위해 `.bat` 파일은 항상 CRLF 줄바꿈으로 보관 — 안 그러면 Windows에서 실행이 깨진다.)

### JSON REST API + 스트리밍(NDJSON)
- **JSON:** 데이터를 **사람도 읽을 수 있게** 적는 표준 형식. `{"이름": "홍길동", "결과": "취약"}` 처럼 *이름:값* 쌍으로 쓴다.
  - 화면(React)과 서버(FastAPI)는 이 JSON으로 **서로 대화**한다.
- **스트리밍 / NDJSON:**
  - 보통은 일을 **다 끝낸 뒤 한 번에** 결과를 준다.
  - 그런데 항목이 100개면 다 끝날 때까지 **몇 분을 빈 화면으로** 기다려야 한다.
  - 그래서 **한 항목 판정이 끝날 때마다 즉시 한 줄씩** 화면에 보내 준다 → "진행 32/100" 처럼 **실시간으로** 보인다.
  - **NDJSON**(Newline-Delimited JSON) = **JSON 한 개를 한 줄씩** 줄바꿈으로 이어 보내는 방식. 스트리밍에 딱 맞는다.
  - 비유: 시험 채점을 **다 끝나고 한꺼번에** 알려주는 대신, **한 문제 채점할 때마다 바로바로** 알려주는 것.

## 한눈에 보는 흐름

```
[사용자]
   │  ① CSV 업로드 (브라우저 화면 = React + Vite + JSX/CSS)
   ▼
[프런트엔드] ──② "판정해 줘" 요청(JSON REST API)──▶ [백엔드: FastAPI + Uvicorn]
                                                          │
                              ③ CSV 정리·집계 (pandas)     │
                              ④ 가이드 PDF 글자 추출 (pypdf) │
                                                          ▼
                              ⑤ 한 항목씩 AI 판정 (claude-agent-sdk → Claude Opus 4.8,
                                 필요하면 WebSearch)
                                                          │
        ◀── ⑥ 판정 결과 한 줄씩 실시간 전송 (스트리밍 NDJSON) ──┘
   │
   ▼
[사용자] ⑦ 검토·수정 후 ⑧ 엑셀 보고서 다운로드 (openpyxl)

   ※ 이 모든 코드는 Git/GitHub로 팀원들과 함께 관리한다.
```

---

> 용어 한 줄 정리
> - **라이브러리/SDK:** 남이 미리 만들어 둔 **부품·도구 모음** (바퀴를 새로 발명하지 않고 가져다 씀)
> - **서버:** 요청을 받아 **일을 처리해 돌려주는 컴퓨터/프로그램**
> - **토큰:** AI가 글을 처리하는 **단위 = 비용**. 적게 쓸수록 싸고 빠르다
> - **빌드:** 사람이 쓴 코드를 **컴퓨터·브라우저가 실행할 형태로 묶는 과정**
